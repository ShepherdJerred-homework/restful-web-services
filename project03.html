Part 1

Model

You are to make a model type for a collection of users.

A User document should have the following fields:

  • username — a string; user's username (e.g., "gfoust")
  • firstname — a string; user's first name (e.g., "Gabriel")
  • lastname — a string; user's last name (e.g., "Foust")
  • email — a string; user's email address (e.g., "gfoust@harding.edu")
  • role — a string; either "admin", "teacher", or "student"
  • salt — a string; salt used to encrypt user's password
  • password — a string; user's encrypted password

Web Services

You should define these services for interacting with User documents:

  • GET /api/users/ — return a list of all users
  • GET /api/users/:username — get a specific user
  • USER /api/users/ — create a new user
  • PUT /api/users/:username — update a specific user
  • DELETE /api/users/:username — delete a specific user

Rules for user web services

  1. When creating or updating, do not allow the client to specify salt; instead generate randomly upon creation and never change
  2. Client will send password unencrypted; encrypt before storing in the database
  3. Web services should never return the salt or password of a user
